Overview

Security Without Third Parties

Most bot detection relies on reCAPTCHA, hCaptcha, or Cloudflare Turnstile — third-party services that add latency, collect user data, and can break on privacy-focused browsers. This system does it completely in-house.

The frontend layer uses behavioural signals: mouse movement entropy, click timing distributions, scroll velocity, and honeypot field interactions. These signals are scored client-side and verified server-side before any extraction request is processed.

The backend layer enforces per-IP, per-endpoint rate limits using an in-memory token bucket store with automatic expiry. A separate age-gate system adds a multi-step CAPTCHA challenge — checkbox verification, emoji image grid, and three confirmation steps — for restricted content access.

Detection Layers

How Bots Get Caught

1
Honeypot Fields
Hidden form fields with display:none — invisible to humans, filled by autobots
Bot → Blocked
2
Mouse Entropy Analysis
Tracks mouse movement variance — bots move in straight lines, humans don't
Bot → Blocked
3
Timing Fingerprint
Measures time-on-page before first interaction — sub-50ms is a bot signal
Bot → Blocked
4
IP Rate Limiting
Token bucket per IP per endpoint — burst allowed, sustained abuse blocked with 429
Abuse → Throttled
5
Age-Gate Multi-Step CAPTCHA
Checkbox → Emoji image grid → 3 confirmation steps — server token required for access
Human → Passed
Features

Technical Depth

🧠
Behavioural Scoring
Client-side JS scores interaction quality (0–100). Requests below threshold are rejected before they reach the extraction pipeline.
🪤
Honeypot Traps
CSS-hidden form fields that are never visible to real users. Any submission with these fields filled is immediately blocked.
⏳
Token Bucket Rate Limiting
Memory-safe token store with automatic cleanup. Separate limits per endpoint — /extract is tighter than /health.
🔢
Emoji Image Grid CAPTCHA
Custom image-based challenge requiring correct selection from a grid of emoji images — defeats simple form bots.
🎟️
Server-Side Token Validation
Age-gate generates a signed server token. Token is invalidated on every new page entry, forcing re-verification.
🛡️
SSRF Hardening
Proxy endpoints validate destination URLs against an allowlist — prevents internal network probing via crafted proxy requests.
