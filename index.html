Philosophy

How I Build

Three rules I don't negotiate on. Every project, every time.

01
Systems before code
I map the problem before touching a keyboard. What breaks? What scales? What's the actual constraint? Architecture decisions made in 10 minutes are worth more than 10 hours of refactoring.
02
Ship early, then sharpen
A working v1 live in 48 hours beats a perfect v3 sitting in a draft folder. I use real behaviour to decide what gets built next — not assumptions. VidSnap's entire feature set came from what people actually tried to do.
03
Own every layer
Backend, frontend, infra, security — I don't hand off. Not because I can't collaborate, but because the best products come from someone who understands the whole stack and the full trade-off at each layer.
Process

How I Think

01
Define the actual problem Not the presented problem. What is the user actually trying to do? Most requests are solutions in disguise. I push back to find the real constraint before designing anything.
02
Find the smallest working proof Can I test the core assumption with 50 lines? I build the thinnest possible slice that proves or kills the idea. VidSnap started as a 3-function Flask endpoint with no UI.
03
Harden by layer Auth, rate limiting, CORS, error states — not as afterthoughts but as deliberate passes. I move through the stack systematically once the core logic is proven.
04
Observe under real load I watch how real users interact before I build anything new. Logs and real traffic patterns replace guesswork entirely. What breaks in production never matches what breaks in dev.
Case Study — VidSnap
The dual-server problem
Free-tier servers cold-start after 15 min idle. First instinct: warm them with pings. Real fix: sequential failover so Server 2 picks up while Server 1 wakes. Users never see a failed request. Zero cost, 2 hours of engineering.
Case Study — Bot Detection
No CAPTCHA library, no third party
Built behavioural analysis from scratch — mouse velocity, timing gaps, honeypot fields. Drops ~85% of automated requests at the edge. No monthly API bill, fully auditable, zero external dependency.
Case Study — HMAC Auth
Stateless verification at the edge
HMAC-signed one-time tokens with expiry timestamps. No database. No session storage. Stateless verification that degrades gracefully and can't be replayed. Implemented in one middleware function.
